# Roadmap Docs

Use this folder for sprint/module planning that expands the main roadmap into implementation-ready tasks.

## Planned Documents

| Document | Purpose |
| --- | --- |
| `mvp-module-plan.md` | Small MVP modules with objective, stories, deliverables, tests, and demos. |
| `production-readiness-plan.md` | Observability, security, resilience, performance, release, and compliance modules. |
| `enterprise-scale-plan.md` | Multi-warehouse, loyalty, analytics, fraud/risk, multi-currency, and compliance reporting. |
| `phase-0.md` | Canonical Phase 0 system design package with boundaries, C4 diagrams, API conventions, data model, auth, security, operations, and production-readiness checklist. |
| `phase-0-discovery-system-design.md` | Compatibility pointer to `phase-0.md`. |
| `phase-1-mvp-foundation.md` | Clean architecture, identity, database, catalog baseline, quality gates. |
| `phase-2-mvp-commerce-flow.md` | Product browsing, cart, inventory reservation, checkout, payment, orders. |
| `phase-3-mvp-experience-layer.md` | Variants, wishlist, reviews, admin dashboard, support tickets. |
| `phase-4-ai-enhanced-mvp.md` | Search index, semantic search, RAG assistants, recommendations. |
| `phase-5-production-readiness.md` | Observability, security, resilience, performance, release, compliance. |
| `phase-6-scale-enterprise.md` | Warehouse, loyalty, analytics, fraud/risk, multi-currency, enterprise operations. |

## Planning Rule

Each module should be small enough that a beginner can implement it by following the checklist, and constrained enough that an AI coding agent does not need to make architecture decisions.
